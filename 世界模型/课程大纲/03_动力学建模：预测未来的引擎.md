## 三、章节大纲

### 🎯 Part 0：导入——天气预报的困境（5min）

**问题导入**：

天气预报员可以比较准确地预测明天的天气，但一周后的预测准确率急剧下降，一个月后几乎毫无意义。

**为什么？**
- 大气系统是混沌系统：微小的初始误差，经过多步传播，变成巨大的最终误差
- 这就是著名的"蝴蝶效应"——这不是预报员的问题，是数学上的必然

**动力学建模面临同样的挑战**：
- 单步预测：给定 s_t, a_t，预测 s_{t+1}——通常准确率较高
- 多步预测：预测 s_{t+H}（H 步之后）——每一步的误差叠加，H 越大误差越大

**为什么我们还是需要多步预测？**
- 强化学习中的规划：要做出好决策，需要预见未来 10-20 步的后果
- 单步预测只能做"贪婪"策略，无法实现长期规划
- 解决方案：学习一个足够好的动力学模型，让多步误差在可接受范围内

### 🌍 Part 1：直觉导入（8min）

#### 1.1 单步 vs 多步预测：乒乓球类比

> **单步预测**：你看着一颗乒乓球飞来，预测下一瞬间它在哪里——速度 + 方向，计算一次即可，很准确。
>
> **多步预测**：预测 10 次弹跳后球在哪里——每次弹跳的角度有微小误差，10 次累积，最终误差可能是球场大小。
>
> **动力学模型的目标**：把每一步的预测误差压到极小，让 10 步、20 步的预测仍然可用。

#### 1.2 "在脑海中模拟"：棋手的世界模型

> 职业围棋手不下棋也能在脑中推演 100 步棋局。他的脑中有一个极其精准的动力学模型：
> - 当前局面（状态 s_t）
> - 一个落子（动作 a_t）
> - 对手的可能回应（随机性：对手可能有多种回应）
> - 下一局面（s_{t+1}）
>
> 他会推演多种"想象的局面"，选择最优的那条线。这正是 DreamerV3 在做的事：在潜空间中推演多步，用 actor-critic 找到最优策略。

#### 1.3 动力学模型的三种类型

| 类型 | 描述 | 适用场景 | 代表 |
|------|------|---------|------|
| **确定性动力学** | s_{t+1} = f(s_t, a_t)，无随机性 | 理想化刚体运动、棋类游戏 | 早期 MBRL 方法 |
| **随机动力学** | s_{t+1} ~ p(s_{t+1} | s_t, a_t)，输出分布 | 有随机噪声的环境、遮挡情况下的不确定性 | 基于 VAE 的方法 |
| **混合动力学（RSSM）** | 分别建模确定性部分（GRU）和随机部分（高斯/类别分布） | 兼顾确定规律和随机性的真实世界 | DreamerV1/V2/V3 |

### 📖 术语表

━━━━━━━━━━━━━━━━━━━━━━

| 术语 | 一句话解释 |
|------|-----------|
| **动力学模型（Dynamics Model）** | 给定当前状态 s_t 和动作 a_t，预测下一状态 s_{t+1} 的模型 |
| **RSSM（循环状态空间模型）** | DreamerV1-V3 的核心：GRU（确定性）+ 分布头（随机性），分别捕捉世界的确定规律和随机因素 |
| **Rollout（展开）** | 在世界模型中执行多步动作，生成想象的状态轨迹 |
| **Model Rollout / Imagination** | 在模型内部（不与真实环境交互）展开多步，用于训练策略 |
| **KL 平衡（KL Balancing）** | DreamerV3 的训练技巧：对先验和后验 KL 损失进行加权，防止模型忽略随机状态 |
| **Free Bits** | KL 损失的下界：只有 KL > 阈值时才计入损失，防止过度正则化 |
| **symlog 变换** | DreamerV3 的值函数技巧：对奖励和价值进行对称对数变换，处理跨任务的奖励尺度差异 |
| **后验（Posterior）** | 已观测 o_t 的条件下，对潜状态 z_t 的估计分布 q(z_t | h_t, o_t) |
| **先验（Prior）** | 仅基于历史（不看 o_t）对潜状态的预测分布 p(z_t | h_t) |
| **世界模型 Rollout 深度（H）** | 想象推演的步数，H 越大规划越长远，但误差累积越多 |

### 🔬 Part 2：颗粒级原理拆解（38min）

#### 2.1 RSSM 架构详解（六层追问）（10min）

| 层 | 问题 | 回答 |
|----|------|------|
| **第 1 层** | RSSM 要解决什么问题？ | 纯确定性动力学模型无法建模不确定性（比如看不到遮挡后的物体）；纯随机模型（帧帧独立的 VAE）无法积累长期依赖。RSSM 用 GRU + 分布头分别处理这两种需求 |
| **第 2 层** | RSSM 的两个状态是什么？ | **确定性状态 h_t**：GRU 隐状态，通过 h_t = GRU(h_{t-1}, z_{t-1}, a_{t-1}) 更新，积累长期依赖；**随机状态 z_t**：从分布 p(z_t | h_t) 或 q(z_t | h_t, o_t) 中采样，表示当前时刻的不确定性 |
| **第 3 层** | 先验和后验的区别？ | **先验** p(z_t | h_t)：在模型内部"凭历史猜测"，用于 Rollout（不看真实观测）；**后验** q(z_t | h_t, o_t)：看到真实观测后的修正估计，用于训练（让先验学会更好地预测后验）。训练目标之一：最小化 KL(posterior ‖ prior） |
| **第 4 层** | 为什么 RSSM 比纯 RNN 好？ | 纯 RNN（LSTM）把所有信息都塞进确定性隐状态，无法区分"确定的规律"和"随机的噪声"。RSSM 显式分离两者：h_t 学确定性规律，z_t 学随机扰动。这让模型更容易学习，也更容易解释 |
| **第 5 层** | RSSM 如何做 Rollout（想象未来）？ | 初始化 h_0, z_0（从真实观测编码）→ 循环：① 给定动作 a_t，更新 h_{t+1} = GRU(h_t, z_t, a_t)；② 从先验采样 z_{t+1} ~ p(z_{t+1} | h_{t+1})；③ 解码预测观测（可选）和奖励 r_{t+1}。H 步后得到完整的想象轨迹 |
| **第 6 层** | RSSM 的训练目标是什么？ | 重建损失（重建 o_t，可选）+ KL 损失（posterior 接近 prior）+ 奖励预测损失。DreamerV3 的关键改进：KL 平衡（后验梯度权重 0.8，先验梯度权重 0.2）防止模式坍塌；free bits（KL 下界 = 1 nat）防止过度正则化 |

**RSSM 数据流图**：

```
真实交互序列：o_1, a_1, o_2, a_2, ..., o_T

训练阶段（有真实观测）：
  o_t ──→ 编码器 ──→ e_t
  h_t, e_t ──→ 后验分布 q ──→ z_t（后验采样）
  h_t, z_t, a_t ──→ GRU ──→ h_{t+1}
  训练信号：KL(q(z_t|h_t,o_t) ‖ p(z_t|h_t)) + recon(o_t) + reward

Rollout 阶段（纯想象，不看真实观测）：
  h_t ──→ 先验分布 p ──→ z_t（先验采样）
  h_t, z_t, a_t ──→ GRU ──→ h_{t+1}
  → 生成想象轨迹，用于 Actor-Critic 训练
```

#### 2.2 DreamerV1：第一个完整 MBRL 框架（8min）

**教学要点**：

| 小节 | 核心内容 |
|------|---------|
| 论文背景 | Hafner et al., ICLR 2020。目标：在 Pixel-based RL 中，学习一个世界模型，在世界模型内部训练 Actor-Critic，减少真实环境交互 |
| 三阶段训练框架 | ① **表示学习**：收集真实数据，训练 RSSM + 编码器（重建 + KL + 奖励）；② **行为学习**：在世界模型的 Rollout 中，用想象轨迹训练 Actor-Critic；③ **环境交互**：用当前策略收集新数据，加入 Replay Buffer |
| Actor-Critic 从"梦"中学习 | Actor：最大化想象 Rollout 中的累计奖励；Critic：估计每一步的价值。关键：反向传播**穿透**世界模型，从 Actor 到 RSSM 参数（梯度 propagation through imagination） |
| DMControl 基准 | 在 DeepMind Control Suite 上，用 5× 更少的真实交互达到 Model-Free SOTA 的水平。样本效率提升是核心贡献 |
| 遗留问题 | 连续潜空间 → 高斯 KL 损失不稳定；Atari 上表现差（部分原因：Atari 需要离散动作 + 游戏图像的特殊性）；跨任务超参数不统一 |

#### 2.3 DreamerV2：离散化的威力（6min）

**关键改进与实验证据**：

| 改进点 | DreamerV1 | DreamerV2 | 实验效果 |
|--------|-----------|-----------|---------|
| 潜空间类型 | 连续（高斯 VAE） | 离散（直通 Softmax，32×32类别） | Atari 55 游戏平均得分 +200% |
| 奖励预测 | MSE 损失 | Twohot-Symlog 分布 | 跨任务奖励尺度更稳定 |
| KL 损失 | 标准 KL | KL 平衡（后验 vs 先验加权） | 防止随机状态坍塌 |
| Transformer | 无 | 无（仍用 GRU） | - |

**为什么离散化在 Atari 上效果更好**：
- Atari 游戏有大量离散的游戏状态（生命值、得分、关卡）——离散表示更自然
- 分类交叉熵损失比 MSE 对异常值更鲁棒
- 32×32 = 1024 个独立的二元特征，更类似人类对场景的"特征化感知"

#### 2.4 DreamerV3：统一超参数的工程艺术（8min）

**DreamerV3 的核心目标**：一套超参数，在所有任务上都工作。

**关键技术贡献**：

| 技术 | 问题 | 解决方案 | 效果 |
|------|------|---------|------|
| **symlog 变换** | 不同任务奖励范围差异巨大（Atari 得分 0-21，Minecraft 奖励 0-1，其他 0-100）→ 无法统一学习率 | 对输入和目标都做 symlog(x) = sign(x)·log(|x|+1)，再做 symlog 回归 | 跨任务奖励尺度不再需要手动归一化 |
| **KL 平衡** | 后验过于确定 → KL 损失主要来自后验，先验得不到充分训练 → 先验预测差 | 对后验梯度乘 0.8，先验梯度乘 0.2（梯度不对称） | 先验预测质量大幅提升，Rollout 更准确 |
| **Free Bits** | 过强的 KL 惩罚 → 模型倾向于忽略随机状态 z（后验=先验=N(0,I)，KL=0） | KL 损失只计算超出阈值（1 nat）的部分 | 随机状态不再坍塌，捕捉真实随机性 |
| **网络规模** | 小模型无法处理复杂任务 | 统一用 RSSM_size=XL（更宽的隐状态），世界模型参数 ~200M | 更强大的表示能力 |

**7 大类任务统一结果（Hafner et al., 2023）**：

#### 2.5 多步预测误差累积分析（4min）

**误差累积的数学直觉**：

假设单步预测相对误差为 ε，H 步后：
- 最坏情况（误差线性叠加）：H·ε
- 实际情况（误差随时间衰减）：介于 ε 和 H·ε 之间，取决于系统的混沌程度

**DreamerV3 的缓解策略**：
- **短 Rollout 深度**：默认 H=15，不做过长推演
- **混合真实+想象数据**：策略梯度来自世界模型，但定期用真实交互校正
- **世界模型持续更新**：随着真实交互增加，世界模型准确性提升
- **离散潜空间的优势**：分类预测误差不会"无限膨胀"，有自然的上界

**开放挑战**：
- 接触丰富的操控任务（机器人抓取）误差累积更快
- 视觉遮挡场景：遮挡后的状态只能靠先验猜测，误差大
- 物理碰撞、流体：动力学高度非线性，难以建模

#### 2.6 🔥 面试考点（2min）

### 💻 Part 3：代码实践（12min）

#### 3.1 简单 RSSM 实现

```python
"""
简化版 RSSM：循环状态空间模型
功能：展示确定性状态（GRU）+ 随机状态（高斯分布）的结合
依赖：torch >= 2.0（pip install torch）
预计运行时间：< 30 秒（CPU）
注意：这是教学级简化版，DreamerV3 的实际 RSSM 更复杂
"""
import torch
import torch.nn as nn
import torch.nn.functional as F
from typing import Tuple, Dict


class SimpleRSSM(nn.Module):
    """
    简化版 RSSM：循环状态空间模型
    论文对应：Hafner et al., DreamerV1/V2/V3

    状态：
      h_t：确定性状态（GRU 隐状态），捕捉确定性时序规律
      z_t：随机状态（高斯分布采样），捕捉随机不确定性
    """
    def __init__(
        self,
        obs_dim: int = 4,       # 观测维度（CartPole = 4）
        action_dim: int = 2,    # 动作维度（CartPole = 2）
        h_dim: int = 64,        # 确定性状态维度
        z_dim: int = 16,        # 随机状态维度
    ):
        super().__init__()
        self.h_dim = h_dim
        self.z_dim = z_dim

        # 确定性转移：h_t = GRU(h_{t-1}, z_{t-1}, a_{t-1})
        self.gru = nn.GRUCell(z_dim + action_dim, h_dim)

        # 先验：p(z_t | h_t)——只看历史，不看当前观测
        self.prior_net = nn.Sequential(
            nn.Linear(h_dim, 32), nn.ELU(),
            nn.Linear(32, z_dim * 2)  # 输出均值 + 对数方差
        )

        # 后验：q(z_t | h_t, o_t)——看到当前观测后的修正
        self.posterior_net = nn.Sequential(
            nn.Linear(h_dim + obs_dim, 64), nn.ELU(),
            nn.Linear(64, z_dim * 2)  # 输出均值 + 对数方差
        )

        # 奖励预测头
        self.reward_head = nn.Sequential(
            nn.Linear(h_dim + z_dim, 32), nn.ELU(),
            nn.Linear(32, 1)
        )

    def get_prior(self, h: torch.Tensor) -> Tuple[torch.Tensor, torch.Tensor]:
        """先验分布 p(z | h)"""
        stats = self.prior_net(h)
        mu, log_var = stats.chunk(2, dim=-1)
        return mu, log_var

    def get_posterior(
        self, h: torch.Tensor, obs: torch.Tensor
    ) -> Tuple[torch.Tensor, torch.Tensor]:
        """后验分布 q(z | h, obs)"""
        stats = self.posterior_net(torch.cat([h, obs], dim=-1))
        mu, log_var = stats.chunk(2, dim=-1)
        return mu, log_var

    def reparameterize(self, mu, log_var):
        std = torch.exp(0.5 * log_var)
        return mu + torch.randn_like(std) * std

    def observe_step(
        self, h: torch.Tensor, obs: torch.Tensor, action: torch.Tensor
    ) -> Dict[str, torch.Tensor]:
        """
        有真实观测时的一步：计算后验，更新确定性状态
        用于世界模型训练（能看到真实 obs）
        """
        # 后验：看到 obs，更新对 z 的估计
        post_mu, post_log_var = self.get_posterior(h, obs)
        z = self.reparameterize(post_mu, post_log_var)

        # 先验：不看 obs，只用历史
        prior_mu, prior_log_var = self.get_prior(h)

        # 更新确定性状态
        action_onehot = F.one_hot(action, num_classes=2).float()
        h_next = self.gru(torch.cat([z, action_onehot], dim=-1), h)

        return {
            'z': z, 'h_next': h_next,
            'post_mu': post_mu, 'post_log_var': post_log_var,
            'prior_mu': prior_mu, 'prior_log_var': prior_log_var,
        }

    def imagine_step(
        self, h: torch.Tensor, z: torch.Tensor, action: torch.Tensor
    ) -> Tuple[torch.Tensor, torch.Tensor]:
        """
        纯想象时的一步：不看真实观测，用先验采样 z
        用于策略训练的 Rollout（不与真实环境交互）
        """
        action_onehot = F.one_hot(action, num_classes=2).float()
        h_next = self.gru(torch.cat([z, action_onehot], dim=-1), h)
        prior_mu, prior_log_var = self.get_prior(h_next)
        z_next = self.reparameterize(prior_mu, prior_log_var)
        return h_next, z_next

    def predict_reward(self, h, z):
        return self.reward_head(torch.cat([h, z], dim=-1)).squeeze(-1)


def demonstrate_rssm():
    """演示 RSSM 的核心操作"""
    import gymnasium as gym

    rssm = SimpleRSSM()
    env = gym.make("CartPole-v1")
    obs, _ = env.reset()
    batch = 1

    h = torch.zeros(batch, rssm.h_dim)
    obs_t = torch.FloatTensor(obs).unsqueeze(0)

    print("=== RSSM 运行演示 ===\n")
    print("【训练模式：有真实观测，用后验估计 z】")
    for step in range(3):
        action = torch.LongTensor([env.action_space.sample()])
        result = rssm.observe_step(h, obs_t, action)
        h = result['h_next']

        kl = 0.5 * torch.mean(
            result['post_mu'].pow(2) + result['post_log_var'].exp()
            - result['post_log_var'] - 1
        ).item()
        print(f"  Step {step+1}: h 范数={h.norm().item():.3f}, "
              f"z 范数={result['z'].norm().item():.3f}, KL={kl:.4f}")

    print("\n【Rollout 模式：纯想象，用先验采样 z（不看真实环境）】")
    z = result['z']
    for step in range(5):
        action = torch.LongTensor([0 if step % 2 == 0 else 1])
        h, z = rssm.imagine_step(h, z, action)
        reward_pred = rssm.predict_reward(h, z).item()
        print(f"  Imagine Step {step+1}: 动作={'左' if action[0]==0 else '右'}, "
              f"预测奖励={reward_pred:.4f}, h 范数={h.norm().item():.3f}")

    env.close()
    print("\n✅ RSSM 演示完成！关键观察：")
    print("  - observe_step 用后验（看真实 obs），imagine_step 用先验（不看 obs）")
    print("  - 这就是 Dreamer 'Dream to Control' 的核心机制")


if __name__ == "__main__":
    demonstrate_rssm()
```

#### 3.2 要点解读

- **两种模式对比**：`observe_step`（训练，有真实观测，用后验）vs `imagine_step`（规划，无真实观测，用先验）
- **KL 损失的含义**：先验 vs 后验的差距，越小说明世界模型的预测越准（不需要观测就能猜到）
- **奖励预测**：基于 (h, z) 预测即将获得的奖励，Actor-Critic 从这个预测信号中学习策略

#### 4.1 动力学建模在产业中的应用

| 应用领域 | 使用的动力学技术 | 核心价值 | 成熟度 |
|---------|--------------|---------|--------|
| 机器人学习（室内） | RSSM + MPC | 减少真实机器人交互，降低损耗 | 研究阶段→早期部署 |
| 自动驾驶预测 | 生成式 + Transformer | 预测行人/车辆未来轨迹 | 生产中（Waymo/Tesla） |
| 游戏 AI（离线学习） | DreamerV3 | 从录像学习，无需在线交互 | 研究成熟 |
| 药物分子设计 | 分子动力学模型 | 在"虚拟化学实验"中筛选候选药物 | 早期应用 |
| 电网调度仿真 | 物理混合模型 | 在仿真中测试调度策略 | 工业应用中 |

#### 4.2 MBRL vs Model-Free：工程选型矩阵

| 场景特征 | 推荐方案 | 原因 |
|---------|---------|------|
| 真实交互代价高（真实机器人、工业控制） | MBRL（DreamerV3/TD-MPC2） | 主要在想象中训练，大幅减少真实样本需求 |
| 仿真便宜、可无限重置（Atari、MuJoCo） | Model-Free 或 MBRL 均可 | MBRL 的样本效率优势减弱，但仍有规划优势 |
| 需要显式规划（多步决策、围棋） | MBRL + MPC/MCTS | 在模型中搜索最优动作序列 |
| 环境高度随机（卡牌游戏、部分观测） | Model-Free 更稳健 | MBRL 的模型误差在高随机性下更难控制 |
| 需要零样本迁移 | MBRL + 结构化先验 | 世界模型可以分享物理先验 |

#### 4.3 前沿动态（2025-2026）

- **TD-MPC2（Hansen et al., 2023）**：另一条 MBRL 路线，用 Temporal Difference 学习和 MPC 结合，在 DMControl 上接近 DreamerV3 性能，推理效率更高
- **视频生成 + 世界模型**：Genie/UniSim 尝试把动力学建模从低维状态空间扩展到高维视频空间，规划在视频帧上进行（计算成本高，但表示更直观）
- **Foundation World Models**：类似 LLM 的预训练世界模型，在大量环境数据上预训练，再迁移到新任务——这是 2025-2026 最热门的研究方向之一

#### 5.2 🔥 面试真题

**真题 1**

> 请描述 Dreamer 系列的三阶段训练框架，并解释每个阶段的目标。

**参考答案框架**：
- **阶段一（表示学习）**：用真实环境数据训练 RSSM + 编码器，目标：让先验能预测后验（KL 最小化），让重建损失和奖励预测损失最小化
- **阶段二（行为学习）**：在 RSSM 的 Rollout 中训练 Actor-Critic，不与真实环境交互，梯度反向传播穿透 RSSM
- **阶段三（数据收集）**：用当前策略与真实环境交互，新数据加入 Replay Buffer，用于阶段一的持续训练
- **关键**：三个阶段交替进行，世界模型和策略相互提升

**真题 2**

> 世界模型在多步 Rollout 中为什么会出现误差累积？有哪些缓解方法？

**参考答案要点**：
- 原因：每步预测误差 → 下一步输入有误差 → 误差逐步叠加，H 步后可能面目全非
- 缓解方法：
  1. 控制 Rollout 深度（DreamerV3 默认 H=15）
  2. 离散潜空间：分类误差有界，不会无限累积
  3. 定期用真实数据重置世界模型（随经验更新）
  4. 集成多个世界模型（MBPO）：用不确定性估计识别高误差区域
  5. 短 Rollout + 准确 Value Function：短期推演 + 长期价值估计的结合
