### 🎯 Part 0：导入——棋手的脑中模拟（5min）

**从棋盘到真实世界**：

| 系统 | 问题 | 解决方式 | 世界模型的角色 |
|------|------|---------|-------------|
| Deep Blue（1997） | 下国际象棋 | 暴力穷举（2 亿步/秒） | 无世界模型，纯搜索 |
| AlphaGo（2016） | 下围棋 | 策略网络 + 价值网络 + MCTS | 价值网络 = 简单世界模型（估计局势） |
| AlphaZero（2017） | 棋类通用 | 自我对弈 + MCTS + 深度学习 | 在"自我对弈想象"中学习 |
| DreamerV3（2023） | 多类 RL 任务 | 在世界模型想象中训练 actor-critic | 完整 RSSM 世界模型 |
| 自动驾驶 FSD | 驾驶 | 规划 + 场景预测 | 预测未来 3-5 秒的场景 |

**规律**：越是样本昂贵、危险的场景，越需要世界模型来"在想象中犯错"。

### 🌍 Part 1：直觉导入（8min）

#### 1.1 规划 vs 反应：两种智能策略

> **反应（Model-Free）**：
> 你走路时，遇到障碍物，你绕开它——这是条件反射式的，不需要"想象"后果。
>
> **规划（基于世界模型）**：
> 你在下象棋，走这步之前，你在脑中想了 3 步：我走这里，对手会走那里，然后我应该……这是规划——在行动之前在脑中预演多步。

| 维度 | 反应（Model-Free） | 规划（MBRL） |
|------|------------------|------------|
| 速度 | 快 | 慢（需要推演） |
| 样本需求 | 多（靠经验） | 少（靠推演） |
| 长期规划 | 弱 | 强 |
| 适合场景 | 快速反射任务（乒乓球） | 需要预判的任务（自动驾驶、棋类） |

#### 1.2 样本效率为什么重要

> **机器人学习的痛苦现实**：
>
> 一台真实机器人学习抓取物品：
> - Model-Free（PPO）：需要约 100 万次真实抓取尝试
> - 100 万次 × 2 秒/次 = 200 万秒 ≈ 23 天
> - 期间：电机磨损、物体损坏、工作人员需要监督
>
> 用世界模型（MBRL）：
> - 前 1000 次真实交互学习世界模型
> - 之后大部分学习在"想象"中进行
> - 真实交互需求降低 10-100 倍

#### 1.3 世界模型让"试错"变成"想象"

**类比：飞行模拟器**

> 飞行员学习前，先在飞行模拟器中训练数百小时。模拟器不是真飞机，但足够逼真，让飞行员掌握核心技能，避免在真实飞机上犯致命错误。
>
> MBRL 的世界模型就是 AI 的"飞行模拟器"——策略在"虚拟环境"中学习，然后迁移到真实世界。

### 📖 术语表

━━━━━━━━━━━━━━━━━━━━━━

| 术语 | 一句话解释 |
|------|-----------|
| **MPC（模型预测控制）** | 在每一步，用世界模型向前推演 H 步，选择使预期回报最大的第一个动作，然后重新规划 |
| **CEM（交叉熵方法）** | MPC 的常用优化器：对动作序列进行采样→评估→取优质样本→拟合新分布→迭代采样 |
| **Dyna 框架** | Sutton 1991 提出的 MBRL 框架：在真实交互和世界模型生成的虚拟经验上同时训练策略 |
| **TD-MPC** | 结合 Temporal Difference 学习和 MPC 的 MBRL 框架；TD-MPC2 是其改进版本 |
| **Rollout 深度（H）** | MPC 或 Dreamer 中向前规划/推演的步数；H 越大规划越长远，计算越贵，误差越大 |
| **样本效率** | 达到同样性能需要多少真实环境交互次数；MBRL 的核心优势 |
| **幻觉规划** | 世界模型误差导致的错误规划：在想象中很好的策略，在真实环境中失败 |
| **迁移差距（Sim-to-Real Gap）** | 在仿真中训练，迁移到真实环境时性能下降的现象 |

### 🔬 Part 2：颗粒级原理拆解（24min）

#### 2.1 MBRL vs Model-Free RL：核心权衡（5min）

**系统级对比**：

| 维度 | Model-Free RL | MBRL |
|------|--------------|------|
| 训练流程 | 直接从真实交互中学习 Q/V 函数或策略 | 先学世界模型，再在模型中训练策略 |
| 样本效率 | 低（PPO 需要数百万步） | 高（可在想象中训练） |
| 模型误差 | 无（直接用真实数据） | 有（世界模型不完美 → 策略可能学错） |
| 算法复杂度 | 低 | 高（需要额外设计世界模型） |
| 规划能力 | 弱（无显式前瞻） | 强（MPC/Rollout 可多步规划） |
| 适合场景 | 仿真便宜（Atari）、环境随机性高 | 真实交互贵（机器人）、需要规划（自动驾驶） |

**世界模型误差的连锁反应**：

```
世界模型误差 ε
    ↓
单步预测误差 ε
    ↓ × H 步
H 步后误差：最坏 H×ε，实际介于 ε 和 H×ε
    ↓
策略在错误的"想象"中学习
    ↓
策略可能对真实环境无效（幻觉规划）
```

**缓解策略**：短 Rollout（DreamerV3 H=15）、混合真实和虚拟数据（Dyna）、不确定性感知规划（只在高置信区域规划）

#### 2.2 MPC：模型预测控制（6min）

MPC 是控制理论中最重要的规划算法，也是世界模型在真实系统中最常用的规划方式。

**MPC 的基本思想**：

```
在时刻 t：
  1. 利用当前状态 s_t 和世界模型，向前规划 H 步
  2. 优化动作序列 [a_t, a_{t+1}, ..., a_{t+H-1}]，使累计回报最大
  3. 只执行第一个动作 a_t
  4. 到时刻 t+1，重新观测真实状态，重新规划（滚动优化）

为什么只执行第一步？
  → 后面几步的规划可能因初始误差而失准
  → 滚动窗口让 MPC 对误差有自我纠正能力
```

**MPC 的优化器：CEM（交叉熵方法）**：

```
输入：当前状态 s_t，世界模型 M，规划深度 H，迭代次数 N_iter

初始化：动作序列分布 a ~ N(0, I)（均匀采样）

重复 N_iter 次：
  1. 从当前分布采样 K 个动作序列 {τ^(1), ..., τ^(K)}
  2. 用世界模型评估每个序列的回报 R(τ^(k))
  3. 取回报最高的 Top-K_elite 个序列
  4. 用这些精英序列更新动作分布（均值和方差）

输出：最优动作序列的第一个动作 a_t
```

**MPC 的优势和局限**：
- 优势：每步都利用最新状态，对模型误差有鲁棒性；可以加入各种约束（安全约束）
- 局限：每步都需要优化，计算成本高；规划深度 H 受限于模型误差累积

#### 2.3 Dyna 框架（4min）

**Dyna（Sutton, 1991）**：第一个系统性的 MBRL 框架

```
Dyna 的三要素：
  1. 真实经验：从真实环境中收集 (s, a, r, s') 四元组
  2. 世界模型：用真实经验训练（或更新）世界模型
  3. 虚拟经验：用世界模型生成额外的 (s, a, r, s') 四元组

每一步真实交互后：
  ① 更新世界模型（用新的 (s, a, r, s')）
  ② 用世界模型生成 K 个虚拟样本
  ③ 用真实 + 虚拟样本更新 Q 函数或策略

直觉：1 步真实经验 → K 步虚拟经验。K 越大，样本效率越高（但世界模型越不准，K 越小越安全）
```

**DreamerV3 和 Dyna 的关系**：

DreamerV3 是深度学习时代的 Dyna——用 RSSM 作为世界模型，在更高维的观测空间（图像）上实现了 Dyna 的核心思想。

#### 2.4 TD-MPC2：当代 MBRL 的高效实现（6min）

**TD-MPC2（Hansen et al., ICLR 2024）**：

| 组件 | 设计 | 与 DreamerV3 对比 |
|------|------|-----------------|
| 表示学习 | 联合嵌入（类 JEPA 思想）+ 时序一致性 | DreamerV3 用 VAE + 离散潜空间 |
| 动力学模型 | 隐式（Latent Dynamics，一步预测） | RSSM（循环，多步） |
| 规划算法 | MPPI（信息论 MPC）/ CEM | Actor-Critic + Rollout |
| 奖励/价值估计 | Q 函数（TD 学习）+ 隐式规划 | 显式奖励头 + Actor-Critic |
| 参数规模 | 较小（~19M） | 较大（~200M） |
| 主要任务 | DMControl 连续控制 | 7 类任务（更广泛） |

**TD-MPC2 的核心贡献**：
- 联合嵌入：不重建像素，只学动作相关的表示（类 JEPA 思想，但用于 RL）
- 更高效的规划：MPPI 比 CEM 更平滑，不需要离散采样
- 规模可扩展：更大的模型在更复杂的任务上性能更好（有 Scaling 特性）

#### 2.5 🔥 面试考点

| 考点 | 标注 | 简要答法 |
|------|------|---------|
| MBRL 和 Model-Free RL 的核心权衡 | 🔥 面试高频 | MBRL：样本效率高，有世界模型误差；Model-Free：样本效率低，无模型误差，更稳定 |
| MPC 为什么每步都重新规划 | 🔥 面试高频 | 滚动优化：利用最新真实观测纠正误差，对世界模型不完美有鲁棒性 |
| Dyna 框架的核心思想 | ⭐ 基础 | 1 步真实经验 + K 步虚拟经验（世界模型生成），同时更新世界模型和策略 |
| TD-MPC2 和 DreamerV3 的主要区别 | ⭐ 加分 | TD 学习 vs Actor-Critic；隐式动力学 vs 显式 RSSM；CEM/MPPI vs Rollout |
| 世界模型在自动驾驶中的作用 | 🔥 面试高频 | 预测其他车/行人的未来轨迹（场景预测）+ 为自车规划安全路径 |

### 💻 Part 3：代码实践（8min）

#### 3.1 简单 MPC 规划器实现

```python
"""
MPC（模型预测控制）+ CEM（交叉熵方法）简单实现
环境：CartPole-v1
世界模型：预训练的简单动力学模型
目标：在不更新任何策略参数的情况下，通过规划控制 CartPole

依赖：torch >= 2.0, gymnasium（pip install torch gymnasium）
预计运行时间：< 1 分钟（CPU）
注意：这是教学级实现，真实 TD-MPC2 要复杂得多
"""
import torch
import torch.nn as nn
import gymnasium as gym
import numpy as np


# ── 1. 简单动力学模型（预训练或在线学习）─────────────────
class DynamicsModel(nn.Module):
    """预测下一状态和奖励"""
    def __init__(self, obs_dim: int = 4, action_dim: int = 2):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(obs_dim + action_dim, 128), nn.ELU(),
            nn.Linear(128, 64), nn.ELU(),
        )
        self.state_head = nn.Linear(64, obs_dim)
        self.reward_head = nn.Linear(64, 1)

    def forward(self, obs, action_onehot):
        x = torch.cat([obs, action_onehot], dim=-1)
        h = self.net(x)
        return self.state_head(h), self.reward_head(h).squeeze(-1)


# ── 2. CEM（交叉熵方法）MPC 规划器 ──────────────────────
class CEMPlanner:
    """
    基于 CEM 的 MPC 规划器
    在世界模型的想象中，优化未来 H 步的动作序列
    """
    def __init__(
        self,
        dynamics: DynamicsModel,
        horizon: int = 10,        # 规划深度 H
        n_samples: int = 256,     # 每次迭代采样数量
        n_elite: int = 32,        # 精英样本数量
        n_iters: int = 5,         # CEM 迭代次数
        n_actions: int = 2,       # 动作数量（CartPole = 2）
    ):
        self.dynamics = dynamics
        self.H = horizon
        self.K = n_samples
        self.K_elite = n_elite
        self.n_iters = n_iters
        self.n_actions = n_actions

    def plan(self, obs: np.ndarray) -> int:
        """给定当前观测，用 CEM 找到最优的第一个动作"""
        obs_tensor = torch.FloatTensor(obs).unsqueeze(0).expand(self.K, -1)

        # 初始化动作序列分布：均匀（每步每个动作等概率）
        # action_probs: [K, H, n_actions]，先用均匀分布采样
        with torch.no_grad():
            for _ in range(self.n_iters):
                # 随机采样动作序列（离散：每步从离散动作中采样）
                actions = torch.randint(0, self.n_actions, (self.K, self.H))

                # 在世界模型中展开 H 步
                total_reward = torch.zeros(self.K)
                current_obs = obs_tensor.clone()

                for t in range(self.H):
                    action_t = actions[:, t]
                    action_onehot = torch.zeros(self.K, self.n_actions)
                    action_onehot.scatter_(1, action_t.unsqueeze(1), 1.0)

                    next_obs, reward = self.dynamics(current_obs, action_onehot)
                    total_reward += reward
                    current_obs = next_obs

                # 取 top-K_elite 的动作序列
                elite_idx = total_reward.topk(self.K_elite).indices
                elite_actions = actions[elite_idx]  # [K_elite, H]

                # ⚠️ 简化：直接用精英中第一步动作的众数作为最优动作
                # 真实 CEM：对精英样本的分布做统计，重新采样
                best_first_action = elite_actions[:, 0].mode().values.item()

        return best_first_action


# ── 3. 快速预训练动力学模型 ────────────────────────────
def pretrain_dynamics(n_steps: int = 5000) -> DynamicsModel:
    """用随机策略数据预训练简单动力学模型"""
    env = gym.make("CartPole-v1")
    model = DynamicsModel()
    optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

    obs, _ = env.reset()
    obs_list, action_list, next_obs_list = [], [], []

    for _ in range(n_steps):
        action = env.action_space.sample()
        next_obs, _, done, truncated, _ = env.step(action)
        obs_list.append(obs)
        action_list.append(action)
        next_obs_list.append(next_obs)
        obs = next_obs
        if done or truncated:
            obs, _ = env.reset()
    env.close()

    obs_t = torch.FloatTensor(obs_list)
    act_t = torch.zeros(len(action_list), 2)
    act_t.scatter_(1, torch.LongTensor(action_list).unsqueeze(1), 1.0)
    next_obs_t = torch.FloatTensor(next_obs_list)

    # 简单奖励：极的角度越小越好（CartPole 目标）
    reward_t = (1.0 - obs_t[:, 2].abs() / 0.2095).clamp(0, 1)

    for _ in range(200):
        pred_next, pred_r = model(obs_t, act_t)
        loss = nn.MSELoss()(pred_next, next_obs_t) + nn.MSELoss()(pred_r, reward_t)
        optimizer.zero_grad(); loss.backward(); optimizer.step()

    return model


# ── 4. 用 MPC 控制 CartPole ───────────────────────────
def run_mpc_demo():
    print("预训练动力学模型...")
    dynamics = pretrain_dynamics(n_steps=5000)

    planner = CEMPlanner(dynamics, horizon=10, n_samples=256, n_elite=32, n_iters=5)

    env = gym.make("CartPole-v1")
    total_rewards = []

    for episode in range(5):
        obs, _ = env.reset()
        episode_reward = 0

        for step in range(500):
            action = planner.plan(obs)          # MPC 规划
            obs, reward, done, truncated, _ = env.step(action)
            episode_reward += reward
            if done or truncated:
                break

        total_rewards.append(episode_reward)
        print(f"  Episode {episode+1}: 总奖励 = {episode_reward:.0f}")

    env.close()
    print(f"\n平均奖励: {np.mean(total_rewards):.1f} (随机策略基线: ~22)")
    print("\n关键洞察：MPC 通过在世界模型中'向前想 10 步'来做决策，")
    print("而不是靠条件反射——这就是规划 vs 反应的本质区别。")


if __name__ == "__main__":
    run_mpc_demo()
```

### 🎓 Part 4：三大应用场景深度拆解（8min）

#### 4.1 自动驾驶：场景预测 + 规划

**自动驾驶世界模型的核心任务**：

```
感知层：Camera/LiDAR → 3D 场景表示
    ↓
世界模型（预测层）：
  输入：当前场景 + 历史轨迹
  输出：未来 3-5 秒的场景预测
    ├── 其他车的轨迹预测（其他 agent 的行为）
    ├── 行人的轨迹预测
    └── 自车执行某个动作后的场景预测
    ↓
规划层（MPC）：
  在预测场景中选择安全、高效的行驶轨迹
```

**代表技术**：

| 公司/系统 | 世界模型方法 | 特点 |
|---------|------------|------|
| Wayve GAIA-1（2023） | 生成式世界模型（Transformer + Diffusion） | 从驾驶视频生成合成驾驶场景，数据增强 |
| Tesla FSD | 神经网络预测 + 规划 | 基于视频的大规模自监督学习 |
| Waymo（研究） | 占用网格预测 + 行为预测 | 结构化场景表示 + 多 agent 预测 |
| UniAD（Hu et al., 2023） | 统一自动驾驶框架 | 端到端：感知→预测→规划统一建模 |

**核心挑战**：
- 长尾分布：罕见但危险的场景（行人突然出现、道路破损）
- 多 agent 交互：预测其他车/行人的行为本质上是在为其他 agent 建立世界模型
- 安全约束：MPC 中需要硬性安全约束（不得超速、不得碰撞）
- 实时性：<100ms 决策，世界模型推理需要极其高效

#### 4.2 游戏 AI：DreamerV3 + MCTS

**游戏 AI 中世界模型的价值**：

| 游戏类型 | 核心挑战 | 世界模型的作用 |
|---------|---------|-------------|
| Atari（完全可观测） | 样本效率 | DreamerV3：在想象中训练，减少真实游戏帧消耗 |
| 棋类（完全信息） | 搜索深度 | MCTS + 价值网络：在树中高效剪枝 |
| 开放世界（部分可观测） | 长期规划 | 维护当前世界状态估计，规划长期策略 |
| 实时策略（不完全信息） | 对手建模 | 对对手行为建世界模型，预测对手动作 |

**DreamerV3 在游戏中的成就**：
- Atari 100K：用 100K 步（约 30 分钟游戏时间）超越 Model-Free 方法用数百万步的成绩
- Crafter（开放世界生存）：首次 AI 在此基准上接近人类水平，需要长期规划
- NetHack：高度随机的 Roguelike 游戏，MBRL 路线首次取得竞争力成绩

#### 4.3 机器人规划：操控 + 导航

**机器人中世界模型的核心挑战**：

| 挑战 | 问题描述 | 当前解决方案 |
|------|---------|------------|
| Sim-to-Real Gap | 仿真中学到的策略迁移到真实机器人性能下降 | 域随机化、真实数据微调 |
| 接触丰富的操控 | 抓取、装配涉及复杂物理接触，难以建模 | 学习基于接触的动力学模型 |
| 长视野规划 | 做饭、整理房间等需要数十步规划 | 分层规划：LLM 做高层规划 + 世界模型做低层执行 |
| 人机协作 | 预测人类意图，为人类行为建世界模型 | 多 agent 世界模型 |

**典型案例：TD-MPC2 + 机器人**：
- 在 DMControl 连续控制任务上 SOTA（机器人手臂、足式机器人）
- 真实机器人实验：比 SAC（Model-Free）少 5-10 倍真实数据就达到同等性能
- 关键技术：隐式动力学模型（不重建像素）+ MPPI 规划

### 📝 Part 6：作业与测试（5min）

#### 6.1 基础题

1. MPC 为什么每一步都要重新规划，而不是一开始规划好整个序列就执行到底？

2. Dyna 框架中，"虚拟经验"（世界模型生成）和"真实经验"各有什么作用？如果世界模型不准确，增加虚拟经验的比例会有什么风险？

3. 世界模型在自动驾驶和游戏 AI 中的使用有什么异同？各面临的核心挑战是什么？

#### 6.2 🔥 面试真题

**真题 1**

> 请设计一个基于世界模型的机器人抓取系统的学习流程。你会如何减少真实机器人的交互次数？

**参考答案框架**：
- 阶段一：收集少量真实交互数据（如 1000 次抓取）
- 阶段二：用真实数据训练视觉 + 动力学世界模型
- 阶段三：在世界模型的"想象"中大量训练抓取策略（Dyna 或 DreamerV3 风格）
- 阶段四：每隔一段时间用真实机器人验证，收集新数据更新世界模型
- 关键设计：不确定性感知的规划（避开世界模型置信度低的区域）；安全约束（MPC 中添加碰撞约束）

**真题 2**

> CEM（交叉熵方法）是如何用于 MPC 规划的？相比于梯度下降，它有什么优势和劣势？

**参考答案要点**：
- CEM 流程：采样 → 评估 → 取精英 → 拟合新分布 → 迭代
- 优势：不需要动力学模型可微分（可用于离散动作/不可微过程）；天然处理多峰分布；避免局部最优（采样式搜索）
- 劣势：需要大量样本（计算成本高）；收敛慢（迭代次数多）；在高维动作空间效率差
- 替代：MPPI（更平滑，利用所有样本加权）；可微 MPC（梯度下降，需要可微动力学）
