## 一、课程定位

### 1.1 面向人群与核心诉求

本课程体系服务 **四类人群**，每类人群有明确的核心诉求：

| 受众 | 占比 | 核心诉求 | 学习场景 |
|------|------|---------|---------|
| **想转行的小白** | ~30% | "AI 是什么？我能不能转行？从哪里开始？" | 下班后自学、周末系统学习 |
| **面试准备者** | ~25% | "面试会考什么？怎么答到点子上？" | 面试前 1-3 个月集中突击 |
| **算法工程师（复习/查漏补缺）** | ~25% | "原理吃透了吗？前沿跟上了吗？" | 工作中遇到问题时查阅、系统性复习 |
| **AI 产品经理** | ~20% | "技术能做什么/不能做什么？怎么和工程师沟通？" | 项目需求评估、技术选型会议前 |

### 1.2 课程独特定位

**不是纯技术课，也不是纯科普课。**

| 对比维度 | 传统技术课 | 传统科普课 | **我们** |
|---------|-----------|-----------|---------|
| 深度 | 很深 | 很浅 | **深浅都有，分层呈现** |
| 受众 | 只看工程师 | 只看大众 | **四类人群各取所需** |
| 讲法 | 公式+代码 | 故事+类比 | **类比→公式→代码→产品，四段递进** |
| 目标 | 学会技术 | 了解概念 | **懂原理 + 会手推 + 能实践 + 明产品价值** |

**一句话定位**：
> 一套面向多类人群的 AI 核心知识体系，让小白能入门，让工程师能过关，让产品经理能决策。

## 三、内容设计原则

### 3.1 核心原则：一个概念，四种讲法

每个核心概念必须从四个层次展开：

```
第 1 层：类比层（小白/产品经理）
  → 用生活中熟悉的事物做类比，建立直觉

第 2 层：原理层（所有人）
  → 公式、算法、架构图，解释"是什么"和"为什么"

第 3 层：代码层（算法工程师）
  → 从零实现 + 框架调用，解释"怎么做"

第 4 层：产品层（产品经理/创业者）
  → 产品应用案例、商业价值、技术选型
```

### 3.2 原则 1：直觉先行，公式后置

**永远不要一上来就写公式。**

#### ❌ 错误示范
```markdown
## 自注意力机制

Attention(Q, K, V) = softmax(QK^T / √d_k) V

其中 Q 是查询向量，K 是键向量...
```

#### ✅ 正确示范
```markdown
## 自注意力机制

### 先理解一个问题

假设你在读一句话："苹果发布了新的 iPhone，它的电池续航很强。"

你怎么知道"它"指的是 iPhone 而不是苹果？

大脑会自动把"它"和"iPhone"关联起来 —— 这就是**注意力**的本质：
**在一大堆信息中，找到和当前最相关的部分。**

### 用图书馆找书来理解

- **Query（查询）** = 你脑子里的搜索词
- **Key（键）** = 每本书的标签
- **Value（值）** = 书里的实际内容

你要做的：
1. 用 Query 去匹配每本书的 Key
2. 根据匹配程度打分
3. 打分高的书，多看它的 Value

### 写成公式

Attention(Q, K, V) = softmax(Q × K^T / √d_k) × V
```

### 3.3 原则 2：每个"为什么"都要回答

技术课程最大的问题不是"讲得太深"，而是"讲得太浅——只讲是什么，不讲为什么"。

#### 必须回答的"为什么"清单

| 类别 | 示例问题 |
|------|---------|
| **设计动机** | 为什么需要这个机制？不用会怎样？ |
| **方案选择** | 为什么用 A 不用 B？A 比 B 好在哪里？ |
| **参数选择** | 为什么除以 √d_k？为什么是 12 个头？为什么 d_ff = 4×d_model？ |
| **结构选择** | 为什么 LayerNorm 放在 Attention 前面？为什么用残差连接？ |
| **历史演进** | 这个设计是怎么演变来的？之前的方案有什么缺陷？ |

### 3.4 原则 3：面试考点显式标注

#### 标注体系

| 标注 | 含义 | 使用场景 |
|------|------|---------|
| 🔥 面试高频 | 近 12 个月内出现在多家公司面试题中 | 必须掌握 |
| ⚠️ 易错 | 大部分人会答错或理解偏差的点 | 特别注意 |
| 💡 面试加分 | 答到这一点会让面试官眼前一亮 | 进阶掌握 |
| 📊 面试实战 | 附带真实面试题 | 练习 |

#### 示例

```markdown
### 为什么除以 √d_k ？

**🔥 面试高频 | ⚠️ 大部分人只能答到第一层**

**第一层（及格线）**：
防止点积值过大导致 softmax 进入饱和区，梯度趋近于 0。

**第二层（良好）**：
当 d_k 增大时，Q 和 K 的点积值的方差会线性增长。
除以 √d_k 使方差保持稳定（≈1），保证 softmax 的梯度有效。

**第三层（加分 💡）**：
这个推导假设 Q 和 K 的每个分量是独立同分布的随机变量，
均值为 0，方差为 1。由中心极限定理，d_k 个这样的变量
的点积的方差为 d_k，所以除以 √d_k 归一化。

📊 真实面试题：
"如果不除以 √d_k，在 d_k=64 和 d_k=1024 时，softmax 的
输出分布会有什么不同？请画图说明。"
```

### 3.5 原则 4：产品视角贯穿始终

每个技术概念都要回答产品经理关心的问题：

#### 产品经理三问

```
1. 这个技术让什么产品成为可能？
2. 如果不用这个技术，产品体验会差在哪里？
3. 我作为产品经理，做需求评审时需要关注什么？
```

#### 示例

```markdown
### 🎓 产品视角：多头注意力的商业意义

**它让什么产品成为可能？**
- ChatGPT 能同时理解一句话中的语法关系、语义关系和指代关系
- 搜索系统能同时理解用户的查询意图、上下文和个性化偏好

**如果不用的后果：**
- 单头注意力只能从一个角度理解句子
- 类比：一个分析师只能从财务角度看公司，看不到技术、市场、运营

**产品经理需要关注的：**
- 多头数影响推理成本（头数翻倍 ≈ 成本增加 30%）
- 如果产品对延迟敏感，可以考虑 GQA（分组查询注意力）
- 和产品确认：精度 vs 速度，哪个更重要？
```

### 3.6 原则 5：对比与关联

#### 横向对比

| 维度 | 方法 A | 方法 B | 怎么选 |
|------|--------|--------|--------|
| 精度 | 高 | 中 | 精度优先选 A |
| 速度 | 慢 | 快 | 延迟敏感选 B |
| 成本 | 高 | 低 | 预算有限选 B |
| 适用场景 | X | Y | 看场景 |

#### 纵向关联

每个知识点都要标注：
- **前置知识**：学这个之前需要先懂什么
- **后续知识**：学完这个之后会用到哪里
- **关联知识**：和哪些概念是平行的

```
前置知识：线性代数基础、神经网络基础
后续知识：GPT 架构、BERT 架构、RAG
关联知识：RNN（对比学习）、CNN（注意力 vs 卷积）
```

### 3.7 原则 6：可视化优先

**一图胜千言。** 每个重要概念都必须有可视化支撑。

#### 可视化类型要求

| 概念类型 | 可视化方式 | 示例 |
|---------|-----------|------|
| 数据流向 | 流程图（箭头标注维度变化） | 自注意力的 Q→K→V 计算流 |
| 结构组成 | 架构图（分层标注） | Transformer Encoder 6 层堆叠 |
| 概念对比 | 对比表 + 对比图 | Attention vs CNN vs RNN |
| 数学推导 | 分步推导（每步标注） | 注意力分数计算步骤 |
| 产品应用 | 产品截图 + 标注 | ChatGPT 的流式输出 |

#### 最小可视化标准

```
每门课程至少包含：
□ 1 张整体架构图
□ 2 张以上细节流程图
□ 1 张对比表/图
□ 所有图都有中文标注
□ 所有图都标注了张量形状
```


### 4.4 知识点颗粒度要求

#### 粒度定义

| 粒度 | 定义 | 示例 |
|------|------|------|
| **粗粒度** | 概念级理解 | "Attention 是让模型关注相关信息" |
| **中粒度** | 原理级理解 | "Attention 通过 Q×K 计算相似度，softmax 归一化" |
| **细粒度** | 公式级理解 | "Attention(Q,K,V) = softmax(QK^T/√d_k)V" |
| **极细粒度** | 推导级理解 | "为什么除以 √d_k？中心极限定理 → 方差归一化" |

#### 各课程最低粒度要求

| 课程 | 最低粒度 | 说明 |
|------|---------|------|
| 1-3（基础层） | **细粒度** | 核心概念必须到公式级 |
| 4-6（应用层） | **中粒度** | 理解原理即可，公式选讲 |
| 7-10（系统层） | **中-细粒度** | 架构到细粒度，使用到中粒度 |
