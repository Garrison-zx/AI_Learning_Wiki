## 一、课程定位

### 1.1 面向人群与核心诉求

本课程体系服务 **四类人群**，每类人群有明确的核心诉求：

| 受众 | 占比 | 核心诉求 | 学习场景 |
|------|------|---------|---------|
| **想转行的小白** | ~30% | "AI 是什么？我能不能转行？从哪里开始？" | 下班后自学、周末系统学习 |
| **面试准备者** | ~25% | "面试会考什么？怎么答到点子上？" | 面试前 1-3 个月集中突击 |
| **算法工程师（复习/查漏补缺）** | ~25% | "原理吃透了吗？前沿跟上了吗？" | 工作中遇到问题时查阅、系统性复习 |
| **AI 产品经理** | ~20% | "技术能做什么/不能做什么？怎么和工程师沟通？" | 项目需求评估、技术选型会议前 |

### 1.2 课程独特定位

**不是纯技术课，也不是纯科普课。**

| 对比维度 | 传统技术课 | 传统科普课 | **我们** |
|---------|-----------|-----------|---------|
| 深度 | 很深 | 很浅 | **深浅都有，分层呈现** |
| 受众 | 只看工程师 | 只看大众 | **四类人群各取所需** |
| 讲法 | 公式+代码 | 故事+类比 | **类比→公式→代码→产品，四段递进** |
| 目标 | 学会技术 | 了解概念 | **懂原理 + 会手推 + 能实践 + 明产品价值** |

**一句话定位**：
> 一套面向多类人群的 AI 核心知识体系，让小白能入门，让工程师能过关，让产品经理能决策。

### 1.3 课程与现有资源的差异化

| 资源 | 优势 | 我们的差异化 |
|------|------|------------|
| 3Blue1Brown | 直觉化动画极强 | 我们补上**代码实现 + 面试考点 + 产品视角** |
| Karpathy | 从零手写代码极强 | 我们补上**小白类比 + 产品经理视角 + 体系化串联** |
| Jay Alammar | 图解极强 | 我们补上**深度问答（为什么）+ 实战代码 + 面试** |
| 吴恩达 | 体系化极强 | 我们补上**最新前沿（2024-2026）+ 中文产品案例** |
| 李宏毅 | 中文+幽默+前沿 | 我们补上**标准化课程模板 + 可复用 + 多人群覆盖** |

## 二、课程体系目标

### 2.1 整体学习目标矩阵

学员完成全部 10 门课程后，应具备以下能力：

#### 技术能力层

| 能力维度 | 目标描述 | 检验方式 |
|---------|---------|---------|
| **概念理解** | 能用自己的话解释每个核心概念 | 口述/写作输出 |
| **公式推导** | 能手推核心公式（注意力、FFN、损失函数等） | 手写推导测试 |
| **代码实现** | 能不依赖框架从零实现核心组件 | 编码测试 |
| **框架使用** | 能熟练使用 Hugging Face 等框架调用模型 | 项目实战 |
| **调试诊断** | 能看懂模型中间结果并定位问题 | 调试任务 |

#### 产品思维层

| 能力维度 | 目标描述 | 检验方式 |
|---------|---------|---------|
| **技术边界认知** | 知道 AI 能做什么、不能做什么 | 案例分析 |
| **技术选型能力** | 能根据需求选择合适的模型/方案 | 方案设计 |
| **产品化思维** | 理解技术如何转化为产品价值 | 产品设计方案 |
| **沟通能力** | 能用通俗语言向非技术人员解释技术 | 口头表达 |

#### 职业发展层

| 能力维度 | 目标描述 | 检验方式 |
|---------|---------|---------|
| **面试通过** | 能通过 AI/算法岗位的常见面试 | 模拟面试 |
| **论文阅读** | 能独立阅读和理解 Transformer 相关论文 | 论文解读作业 |
| **前沿跟进** | 知道当前研究热点和发展方向 | 开放讨论 |

### 2.2 课程之间的递进关系

```
核心基础层（课程 1-4）
├── 1. Transformer 架构 —— 一切的基础
├── 2. BERT 和 T5 —— Encoder-only 和 Text-to-Text 的代表
├── 3. 大模型基础知识 —— 架构演进、Scaling Laws
└── 4. 模型后训练 —— 微调 + RLHF + 强化学习

├── 6. 模型应用选型 + 部署 —— 怎么选、怎么部署

系统架构层（课程 8-10）
├── 8. Agent 原理与多架构 —— 让大模型能自主行动
├── 9. AI 编程助手：Claude Code / Codex / CLI 生态 —— MCP + Skills
└── 10. OpenClaw 架构 —— 个人 AI 助手框架
```

**递进规则（描述性标签）**：

| 层级 | 标签 | 课程 | 说明 |
|------|------|------|------|
| 核心基础层（课程 1-4） | **核心基础** | Transformer / BERT&T5 / 大模型基础 / 后训练 | 构成知识底座，所有课程的起点 |
| 应用拓展层（课程 5-7） | **应用拓展** | 提示词+Harness / 选型部署 / RAG+检索 | 在基础上学会使用、选型和部署 |
| 系统架构层（课程 8-10） | **系统架构** | Agent 多架构 / 编程助手生态 / OpenClaw | 构建完整的 AI 系统 |

**不同人群对各层的意义**：

| 人群 | 核心基础 | 应用拓展 | 系统架构 |
|------|---------|---------|---------|
| 小白 | 建立知识框架 | 知道 AI 能做什么 | 理解 AI 系统全貌 |
| 面试 | 面试核心考点 | 场景题来源 | 架构设计题来源 |
| 工程师 | 原理吃透 | 工作实战 | 系统设计能力 |

> ⚠️ 注意：不使用"必学/选学/进阶"这种带有价值判断的标签，
> 避免误导某类人群跳过他们实际需要学习的内容。

## 三、内容设计

### 3.1 核心原则

#### 原则 1：一个概念，四种讲法

每个核心概念必须从四个层次展开：

| 层次 | 面向 | 方法 |
|------|------|------|
| 第 1 层：类比层 | 小白/产品经理 | 用生活中熟悉的事物做类比，建立直觉 |
| 第 2 层：原理层 | 所有人 | 公式、算法、架构图，解释"是什么"和"为什么" |
| 第 3 层：代码层 | 算法工程师 | 从零实现 + 框架调用，解释"怎么做" |
| 第 4 层：产品层 | 产品经理/创业者 | 产品应用案例、商业价值、技术选型 |

#### 原则 2：直觉先行，公式后置

**永远不要一上来就写公式。** 先用类比建立直觉，再推导公式。

❌ 错误：上来就写 `Attention(Q, K, V) = softmax(QK^T / √d_k) V`

✅ 正确：先讲"苹果发布了新 iPhone，'它'指的是谁？" → 用图书馆找书类比 Q/K/V → 最后写公式

#### 原则 3：追问到底（六层追问法）

技术课程最大的问题不是"讲得太深"，而是"讲得太浅——停在表面，不再追问"。

每个核心概念必须经过至少 5 层追问，直到抵达底层原理并看清演进脉络：

```
第 1 层：它是什么？        → 概念定义
第 2 层：为什么需要它？     → 设计动机（解决了什么问题？不用会怎样？）
第 3 层：为什么这么设计？   → 方案选择（为什么用 A 不用 B？背后的权衡？）
第 4 层：为什么参数是这样？ → 技术细节（为什么除以 √d_k？为什么是 12 个头？）
第 5 层：底层原理是什么？   → 数学/物理基础（中心极限定理？信息论？计算复杂度？）
第 6 层：它怎么演变来的？往哪里去？ → 演进脉络（前身方案 → 当前方案 → 局限性 → 未来方向）
```

**第 6 层要回答的问题：**
技术不是凭空出现的，每个方案都有一条演进链：**问题出现 → 早期方案 → 暴露缺陷 → 改进方案 → 当前最优 → 未来方向**。

| 追问 | 内容 |
|------|------|
| 之前用的是什么？ | 前身方案及其局限性 |
| 演进路线是什么？ | 时间线 + 关键论文/产品 |
| 当前方案的局限性？ | 还有什么没解决的问题 |
| 下一代方案在探索什么？ | 学术界/工业界的前沿方向 |

**追问示例：位置编码的演进（第 6 层示范）**

| 阶段 | 方案 | 年份 | 解决了什么 | 还有什么问题 |
|------|------|------|-----------|------------|
| 诞生 | 正弦位置编码 | 2017 | 固定编码，无需学习 | 不含相对位置信息，外推差 |
| 改进 | 可学习位置编码（GPT） | 2018 | 自适应学习 | 外推更差 |
| 突破 | RoPE | 2021 | 天然相对位置 + 外推 | 超长上下文衰减 |
| 修补 | NTK-aware / YaRN | 2023 | 扩展到 128K+ | 工程复杂度增加 |
| 前沿 | 无位置编码探索 | 2024+ | 完全去掉？ | 实验阶段 |

**追问示例：Attention（完整 6 层）**

| 层 | 问题 | 回答 |
|----|------|------|
| 1 | 是什么？ | 在序列中找最相关的部分 |
| 2 | 为什么需要？ | RNN 无法处理长程依赖 |
| 3 | 为什么 Q×K 点积？ | 衡量相似度，计算高效 |
| 4 | 为什么除以 √d_k？ | 防 softmax 饱和 |
| 5 | 数学推导？ | 点积方差 = d_k，除以 √d_k 归一化 |
| 6 | 怎么演变来的？ | 加性注意力(2014) → 点积(2015) → 自注意力(2017) → MQA(2019) → GQA(2023) → Flash Attention(2022) → 线性注意力/Mamba |

**追问示例：LayerNorm（完整 6 层）**

| 层 | 问题 | 回答 |
|----|------|------|
| 1 | 是什么？ | 对每个 token 的特征做归一化 |
| 2 | 为什么需要？ | 防止激活值爆炸/消失 |
| 3 | 为什么 LN 不用 BN？ | 序列长度不固定，batch 内分布不稳定 |
| 4 | 为什么放 Attention 前面？ | Pre-LN 梯度可直接绕过 Attention 层 |
| 5 | 底层原理？ | 残差网络信号传播理论，Pre-LN 使方差稳定 |
| 6 | 怎么演变来的？ | BN(2015) → LN(2016) → RMSNorm(2019) → 无归一化探索 |

**必须回答的"为什么"清单（检查项）：**

| 类别 | 示例问题 |
|------|---------|
| 设计动机 | 为什么需要这个机制？不用会怎样？ |
| 方案选择 | 为什么用 A 不用 B？ |
| 参数选择 | 为什么除以 √d_k？为什么 12 个头？ |
| 结构选择 | 为什么 LayerNorm 放在 Attention 前？ |
| 底层原理 | 背后的数学/物理基础？ |
| 演进脉络 | 前身→当前→局限→未来方向？ |

#### 原则 4：产品视角贯穿

每个技术概念都要回答产品经理三问：
1. 这个技术让什么产品成为可能？
2. 不用这个技术，产品体验差在哪？
3. 做需求评审时需要关注什么？

#### 原则 5：对比与关联

**横向对比**：同类方案从精度/速度/成本/场景维度对比。
**纵向关联**：每个知识点标注前置知识、后续知识、关联知识。

### 3.3 体验策略

#### Wow Moment 设计

每门课程至少有一个"顿悟体验"（💥 本课的 Wow Moment），四种类型：

| 类型 | 方法 | 示例 |
|------|------|------|
| 反直觉 | 说出与直觉相反的事实 | "Attention 是 O(n²)，所以 GPT-4 处理长文本其实很慢" |
| 一图看懂 | 一张图解释 10 分钟文字 | Q×K 矩阵乘法图让注意力瞬间清晰 |
| 类比暴击 | 极端贴切的类比 | "LayerNorm 就像每个句子自己给自己打分" |
| 历史反转 | 揭示演进的意外 | "Attention Is All You Need 标题其实是反讽" |

#### 认知负荷控制

| 元素 | 限制 |
|------|------|
| 纯文字段落 | ≤ 5 行 |
| 代码段 | ≤ 40 行 |
| 连续阅读 | ≤ 10 分钟后插入互动 |

#### 互动设计

每 10 分钟至少一次互动：思考题、预测题、对比题、找错题、填空题。

### 4.1.5 术语表规范（课前必放）

**位置**：Part 1（直觉导入）和 Part 2（原理拆解）之间。

**为什么放在这里**：
- Part 1 用类比建立直觉，不需要术语
- Part 2 开始使用专业词汇，必须先对齐认知
- 表格格式让老手一眼扫过就跳过

**标准格式**：

```markdown
━━━━━━━━━━━━━━━━━━━━━━
📖 本课术语表（新手必看，老手跳过）

| 术语 | 一句话解释 | 示例 |
|------|-----------|------|
| 2B / 4B / 32B | 模型参数量，B=十亿（Billion） | 2B=轻量，手机可跑；32B=重量，需多张 GPU |
| token | 模型读文字的最小单位 | "Hello world" = 2 个 token；"你好世界" = 2-4 个 |
| context window | 模型一次能"记住"的最大字数 | 8K ≈ 6000 汉字；128K ≈ 10 万字（一本书） |
| temperature | 控制输出的随机程度 | 0.1=确定；0.7=创意；1.0=发散 |
| ... | ... | ... |

━━━━━━━━━━━━━━━━━━━━━━
```

**编写规则**：
1. 每门课 **5-12 个术语**，不超过一页
2. 解释必须**一句话**（不超过 20 字）+ 具体示例
