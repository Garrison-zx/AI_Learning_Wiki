## Part 1：直觉导入

### 人类手臂运动 → 机器人运动学

**类比：打台球**

打台球时，你不是在解方程。你的大脑接收到"球在哪里"的信息，直接输出"手臂应该怎么动"的指令。这是人类的运动控制系统——经过亿万年进化和数年学习优化的系统。

**两种思考方式：**

| 方式 | 描述 | 类比 |
|------|------|------|
| 正运动学（FK） | 已知关节角度 → 求末端位置 | 知道每个骨骼的角度，求手在哪里 |
| 逆运动学（IK） | 已知末端目标位置 → 求关节角度 | 想把手放到某处，求需要怎么弯曲关节 |

**为什么IK比FK难得多？**
- FK：一个关节角度配置 → 唯一的末端位置（确定性映射）
- IK：一个末端位置 → **多个可能的关节角度配置**（一对多，可能无解）

## Part 2：颗粒级原理拆解

### 2.1 正运动学（Forward Kinematics）

**DH参数法（Denavit-Hartenberg）：**

用4个参数描述相邻关节之间的几何关系：

| 参数 | 符号 | 含义 |
|------|------|------|
| 连杆长度 | a | 沿 x 轴的距离 |
| 连杆偏距 | d | 沿 z 轴的距离 |
| 连杆扭角 | α | 绕 x 轴的转角 |
| 关节角 | θ | 绕 z 轴的转角（可变量） |

**变换矩阵（4×4齐次变换）：**

```
T = Rz(θ) · Tz(d) · Tx(a) · Rx(α)

最终末端位置 = T₁ · T₂ · T₃ · ... · Tₙ · 原点
```

**ML背景学员**：你可以把正运动学理解为一个确定性的、可解析求导的函数 f(θ) = p，其中 θ 是关节角度向量，p 是末端执行器位置。这个函数后来被用于可微仿真。

### 2.2 逆运动学（Inverse Kinematics）

**三种求解方法：**

**方法1：解析法（Analytic IK）**
- 手工推导代数方程
- 优点：速度极快（微秒级）
- 缺点：只对特定构型有效，需要数学推导

**方法2：数值法（Numeric IK，Jacobian-based）**

```python
# 伪代码：基于雅可比矩阵的迭代IK
while ||target - current_pos|| > threshold:
    J = compute_jacobian(current_joints)  # 雅可比矩阵
    delta_p = target - current_pos        # 位置误差
    delta_theta = J_pseudo_inverse * delta_p  # 最小二乘解
    current_joints += alpha * delta_theta  # 更新关节角度
    current_pos = forward_kinematics(current_joints)
```

- 优点：通用，处理任意构型
- 缺点：迭代计算，奇异点附近失效

**奇异位形（Singularity）**：雅可比矩阵秩亏，机械臂失去某个方向的自由度。实际中必须检测和避免。

**方法3：学习方法（Learning-based IK）**
- 用神经网络直接学习 p → θ 的映射
- 优点：可处理奇异点，天然多解处理
- 缺点：精度有限，需要训练数据
- 代表工作：IKFlow（2022）

### 2.3 关节空间 vs 任务空间

**关键概念对比：**

| 维度 | 关节空间（Joint Space） | 任务空间（Task Space） |
|------|------------------------|----------------------|
| 描述方式 | 关节角度向量 θ ∈ Rⁿ | 末端执行器位姿 (p, R) |
| 直觉 | "肩关节转30度" | "手放到(0.5, 0, 0.3)" |
| 规划难度 | 插值简单 | 插值需要IK求解 |
| 碰撞检测 | 需要转换 | 直接检测 |
| 学习方法输出 | 关节角度/速度 | 末端位姿（可再转换） |

**ML背景学员**：大多数VLA模型（RT-2/π₀）输出的是**关节角度**（关节空间），而不是末端位姿。了解两者的关系有助于理解模型输出。

### 2.4 阻抗控制（Impedance Control）

**为什么刚性位置控制不够？**

> 想象用铁棍推鸡蛋 vs 用弹簧推鸡蛋。铁棍（位置控制）不允许任何偏差，一旦接触就破；弹簧（阻抗控制）允许力和位置的权衡。

**阻抗控制方程：**

```
M·ẍ + B·ẋ + K·(x - x_desired) = F_external

M：虚拟惯性矩阵（惯性响应）
B：虚拟阻尼矩阵（阻尼特性）
K：虚拟刚度矩阵（弹性特性）
F_external：外部接触力
```

**调参直觉：**
- K 大（高刚度）：精确跟踪位置，但接触力大 → 适合非接触任务
- K 小（低刚度）：允许力偏差，顺应接触 → 适合需要接触力控制的任务
- B 大（高阻尼）：运动平滑，但响应慢
- B 小（低阻尼）：响应快，但可能振荡

**在具身智能中的地位**：即使使用端到端学习策略，底层控制器通常仍然是阻抗控制器——学习层产生期望阻抗参数或期望轨迹，控制层执行。

### 2.5 轨迹规划与DMP

**轨迹规划的两个层次：**

```
任务层规划：从A到B的路径（避障/优化）
运动层规划：如何平滑地执行这条路径（速度/加速度剖面）
```

**梯形速度剖面：**
```
加速段 → 匀速段 → 减速段
特点：位置连续，速度连续，加速度有突变
适用：大多数工业场景
```

**动力学运动基元（DMP，Dynamic Movement Primitives）：**

DMP是连接传统控制与学习方法的桥梁：

```python
# DMP的核心方程
τ·ẍ = α_z·(β_z·(g - x) - ẋ) + f(s)

# 其中：
# g：目标位置（可变）
# x：当前位置
# f(s)：强迫项（从示范中学习的部分）
# τ：时间缩放参数

# f(s) 的学习：
# 从人类示范轨迹中，通过回归学习基函数权重
# f(s) = Σ wᵢ · ψᵢ(s) / Σ ψᵢ(s) · (g - x₀)
```

**DMP的核心优势：**
- 可以从少量示范（甚至1次）中学习轨迹
- 轨迹可以缩放（时间/空间缩放）
- 可以用RL微调 wᵢ 参数

**与深度学习的关系：**
- DMP是Diffusion Policy之前的"老派"模仿学习方法
- 现在通常被Diffusion Policy/ACT取代
- 但DMP的思想（轨迹=基函数的组合）影响了后续工作

### 2.6 面试考点

**Q：什么是机器人的奇异位形（Singularity）？如何避免？**

奇异位形：机械臂在某些特定构型下，雅可比矩阵秩亏，导致失去某个方向的运动能力。

避免方法：
1. 工作空间规划（避免经过奇异点）
2. 阻尼最小二乘IK（DLS-IK）
3. 冗余度利用（零空间运动）

**Q：关节空间插值和任务空间插值有什么区别？各自适用场景？**

- 关节空间插值：计算简单，但末端轨迹不直观（可能走弯路）
- 任务空间插值：末端轨迹可预测（直线），但需要连续IK求解
- 精密装配用任务空间；一般运动用关节空间

**Q：阻抗控制的物理意义是什么？与纯位置控制有何不同？**

阻抗控制定义了力和位置偏差之间的关系（像弹簧-阻尼系统），允许柔顺地与环境接触。纯位置控制强制跟踪期望轨迹，接触时可能产生极大的接触力。

## Part 3：代码实践

### 目标：PyBullet中实现逆运动学控制与轨迹规划

```python
import pybullet as p
import pybullet_data
import numpy as np
import time

class RobotController:
    """
    机器人控制器：演示关节空间与任务空间的轨迹规划
    """

    def __init__(self):
        self.client = p.connect(p.GUI)
        p.setGravity(0, 0, -9.81)
        p.setAdditionalSearchPath(pybullet_data.getDataPath())
        p.loadURDF("plane.urdf")
        self.robot = p.loadURDF("kuka_iiwa/model.urdf",
                                basePosition=[0, 0, 0],
                                useFixedBase=True)
        self.n_joints = 7
        self.ee_index = 6

    def forward_kinematics(self, joint_angles):
        """正运动学：给定关节角度，返回末端位置"""
        for i, angle in enumerate(joint_angles):
            p.resetJointState(self.robot, i, angle)
        state = p.getLinkState(self.robot, self.ee_index)
        return np.array(state[0]), np.array(state[1])

    def inverse_kinematics(self, target_pos, target_orn=None):
        """逆运动学：给定末端目标位置，返回关节角度"""
        if target_orn is not None:
            ik_solution = p.calculateInverseKinematics(
                self.robot, self.ee_index, target_pos, target_orn)
        else:
            ik_solution = p.calculateInverseKinematics(
                self.robot, self.ee_index, target_pos)
        return np.array(ik_solution[:self.n_joints])

    def joint_space_trajectory(self, start_angles, end_angles, steps=100):
        """
        关节空间轨迹规划：两组关节角度之间的线性插值
        优点：计算简单
        缺点：末端轨迹不是直线
        """
        trajectory = []
        for t in np.linspace(0, 1, steps):
            # 使用三次多项式插值（比线性更平滑）
            s = 3*t**2 - 2*t**3   # 平滑步进函数
            angles = (1-s) * np.array(start_angles) + s * np.array(end_angles)
            trajectory.append(angles)
        return trajectory

    def task_space_trajectory(self, start_pos, end_pos, steps=100):
        """
        任务空间轨迹规划：末端执行器空间中的直线插值
        优点：末端轨迹是直线，直观可预测
        缺点：需要连续IK求解
        """
        trajectory = []
        for t in np.linspace(0, 1, steps):
            s = 3*t**2 - 2*t**3  # 平滑步进函数
            pos = (1-s) * np.array(start_pos) + s * np.array(end_pos)
            joint_angles = self.inverse_kinematics(pos)
            trajectory.append(joint_angles)
        return trajectory

    def execute_trajectory(self, trajectory, control_freq=240):
        """执行关节角度轨迹"""
        dt = 1.0 / control_freq
        for joint_angles in trajectory:
            for i, angle in enumerate(joint_angles):
                p.setJointMotorControl2(
                    self.robot, i,
                    p.POSITION_CONTROL,
                    targetPosition=angle,
                    force=500,
                    maxVelocity=2.0
                )
            p.stepSimulation()
            time.sleep(dt)

    def compare_trajectories(self):
        """演示：比较关节空间和任务空间轨迹的区别"""

        start_pos = [0.4, 0.2, 0.5]
        end_pos = [0.4, -0.2, 0.5]

        # 获取起始关节角度
        start_angles = self.inverse_kinematics(start_pos)
        end_angles = self.inverse_kinematics(end_pos)

        print("=== 关节空间轨迹 ===")
        joint_traj = self.joint_space_trajectory(start_angles, end_angles)
        # 记录末端轨迹
        ee_positions_joint = []
        for angles in joint_traj:
            pos, _ = self.forward_kinematics(angles)
            ee_positions_joint.append(pos)

        print("=== 任务空间轨迹 ===")
        task_traj = self.task_space_trajectory(start_pos, end_pos)
        ee_positions_task = []
        for angles in task_traj:
            pos, _ = self.forward_kinematics(angles)
            ee_positions_task.append(pos)

        # 分析轨迹直线度
        ee_joint = np.array(ee_positions_joint)
        ee_task = np.array(ee_positions_task)

        # 计算偏离直线的程度
        ideal_line = np.linspace(start_pos, end_pos, len(joint_traj))
        joint_deviation = np.mean(np.linalg.norm(ee_joint - ideal_line, axis=1))
        task_deviation = np.mean(np.linalg.norm(ee_task - ideal_line, axis=1))

        print(f"关节空间：末端平均偏离直线 {joint_deviation:.4f}m")
        print(f"任务空间：末端平均偏离直线 {task_deviation:.4f}m")

        return joint_traj, task_traj


if __name__ == "__main__":
    controller = RobotController()
    joint_traj, task_traj = controller.compare_trajectories()

    print("\n执行任务空间轨迹（末端走直线）...")
    controller.execute_trajectory(task_traj)
    time.sleep(1)
    p.disconnect()
```

## Part 4：产品视角

### 运动控制能力边界与产品定位

**不同控制方式的能力边界：**

| 控制方式 | 精度 | 泛化性 | 开发成本 | 典型应用 |
|----------|------|--------|----------|----------|
| 固定轨迹控制 | 最高 | 最低 | 低 | 焊接/喷涂等重复任务 |
| 阻抗控制+IK | 高 | 中 | 中 | 精密装配 |
| 学习轨迹（DMP） | 中 | 中 | 中 | 柔性制造 |
| 端到端学习（VLA） | 中 | 高 | 高 | 通用操作 |

**产品经理的核心判断**：

> 不要为了"看起来先进"选择端到端学习。固定场景、高重复率任务，传统控制更可靠、更经济。学习方法的优势在于泛化性——当场景变化时，传统方法需要重新开发，学习方法理论上可以适应。
