# EI-02 具身感知：机器如何感受世界

---

## Wow Moment

> **机器人的手指可能比人类更灵敏——GelSight能感受到0.1mm的纹理变化，而人类指尖的触觉分辨率约为1mm。但拥有超级触觉的机器人，仍然不知道"丝滑"是什么感觉。**

感知不是越精确越好，而是要**为行动服务**。一个装配工厂的机器人需要的是"螺丝孔在哪"，而不是"螺丝孔的材质多么光滑"。具身感知的核心问题是：**如何获取恰好足够且实时的信息，驱动正确的行动？**

---

## Part 0：为什么纯视觉不够？

### 核心问题

GPT-4V能识别图片中的一切。那么，给机器人装一个摄像头，问题解决了吗？

**三个纯视觉无法处理的场景：**

**场景1：倒液体**
- 视觉可以看到杯子，但无法感知倒了多少——你需要重量传感器
- 液体晃动时的预测控制，需要本体感觉

**场景2：拧螺丝**
- 视觉能看到螺丝，但无法感知力矩是否到位
- 拧过头 → 螺纹损坏；拧不够 → 连接不牢
- 精确力控需要力/力矩传感器（FT Sensor）

**场景3：抓鸡蛋**
- 视觉知道鸡蛋的位置，但抓取力度多少才合适？
- 太轻 → 鸡蛋掉落；太重 → 鸡蛋破裂
- 触觉反馈（接触力感知）是必要的

**核心论点**：具身感知是多模态的。单一传感器无论多精确，都无法覆盖所有必要的信息维度。

## Part 1：直觉导入

### 人类多传感融合

人类完成"端一杯热咖啡"这个动作时，使用了哪些感知通道：

| 感知通道 | 感知内容 | 对应机器人传感器 |
|----------|----------|-----------------|
| 视觉 | 杯子位置、装填程度 | RGB/RGB-D摄像头 |
| 触觉 | 握力、杯子温度 | 触觉阵列/温度传感器 |
| 本体感觉 | 手臂位置、倾斜角度 | 关节编码器/IMU |
| 前庭觉 | 身体平衡状态 | IMU |
| 听觉 | 液体晃动声 | 麦克风（通常忽略） |

**关键发现**：人类的每个感知通道都在为特定的行动目标服务，不是收集所有信息。

### 机器人的传感器生态

```
机器人感知层级

外感知（感知外部世界）
├── 视觉感知
│   ├── RGB摄像头：颜色/纹理/形状
│   ├── 深度摄像头（RGB-D）：3D几何
│   ├── 激光雷达（LiDAR）：精确3D点云
│   └── 立体视觉：双目深度估计
│
├── 触觉感知
│   ├── 力/力矩传感器（FT Sensor）：末端6维力
│   ├── 触觉阵列（GelSight/DIGIT）：接触分布
│   └── 皮肤传感器：全身接触
│
└── 其他外感知
    ├── 麦克风阵列：声音定位
    └── 温度传感器

本体感知（感知自身状态）
├── 关节编码器：关节角度/速度
├── IMU：加速度/角速度
└── 电流传感器：关节力矩估计
```

## Part 2：颗粒级原理拆解

### 2.1 RGB-D视觉感知

**RGB-D摄像头工作原理（以 Intel RealSense 为例）：**

```
RGB摄像头 → 颜色信息（480×640×3）
深度传感器 → 深度图（480×640×1，单位：mm）
    ↓
融合 → RGBD图像（480×640×4）
    ↓
点云转换 → 三维点集合（N × 3）
```

**深度感知的三种技术路线：**

| 技术 | 原理 | 代表产品 | 适用场景 | 局限 |
|------|------|----------|----------|------|
| 结构光 | 投影已知图案，通过变形计算深度 | RealSense D435, Kinect | 近距离（<3m）精度高 | 室外强光干扰 |
| ToF（飞行时间） | 测量光脉冲往返时间 | RealSense L515 | 中等距离，实时性好 | 精度相对低 |
| 双目立体视觉 | 两个摄像头视差计算深度 | ZED2, Realsense D435（也有） | 灵活，成本低 | 无纹理区域失效 |
| 激光雷达（LiDAR） | 旋转激光测距 | Velodyne, Ouster | 远距离，高精度 | 成本高，帧率低 |

**ML背景学员关注点**：RGB-D数据的维度远高于普通图片，常见处理方式有两种：
1. 深度图当作额外通道（RGBD → 4通道2D卷积）
2. 转换为点云 → PointNet/PointTransformer处理

**机器人背景学员关注点**：深度摄像头需要标定（内参+外参），且与机器人坐标系的变换关系（手眼标定）是工程实践的关键步骤。

### 2.2 触觉传感：具身感知的关键缺口

**为什么触觉是当前最重要的缺口？**

> "视觉告诉你物体在哪里，触觉告诉你操作是否成功。"

**GelSight（MIT 2009，持续演进）：**
- 原理：橡胶凝胶+背光+摄像头，将接触压力转化为图像
- 分辨率：~0.1mm（优于人类指尖）
- 输出：接触区域3D形貌图
- 应用：检测接触面法向力分布，判断抓取稳定性

**DIGIT（Meta/Facebook AI 2020）：**
- 基于GelSight原理，商业化设计
- 更紧凑，适合集成到夹爪
- 开源设计，已被多个研究团队采用

**触觉数据的特殊性：**
- 数据稀缺（没有互联网级别的触觉数据）
- 传感器间差异大（不同个体传感器性能不一致）
- 与视觉数据的时序对齐是挑战

### 2.3 本体感觉

**关节编码器（Encoder）：**

```
增量式编码器：测量相对角度变化（相对编码）
    → 上电需要归零操作
    → 成本低，分辨率高

绝对式编码器：直接输出绝对角度（绝对编码）
    → 上电即知道位置
    → 成本高，是工业机器人标配
```

**IMU（惯性测量单元）：**
- 加速度计（3轴）：测量线加速度
- 陀螺仪（3轴）：测量角速度
- 融合输出：姿态估计（Roll/Pitch/Yaw）
- 局限：存在漂移，需要定期矫正

**电流/力矩传感：**
- 通过电机电流间接估计关节力矩
- 成本低但精度有限
- 高精度力控需要专用力矩传感器

### 2.4 多传感融合（Sensor Fusion）

**为什么融合比单一传感器好？**

卡尔曼滤波器的核心思想：

```
预测步骤：
    基于运动模型预测下一时刻状态
    状态估计误差 → 增大

更新步骤：
    新的传感器数据到来
    根据传感器噪声特性，加权融合预测和观测
    状态估计误差 → 减小
```

**实际应用：视觉+本体感觉融合（VIO）**

```
IMU（1000Hz）：短期预测精确，长期有漂移
摄像头（30Hz）：绝对位置精确，短期噪声大
    ↓
视觉-惯性里程计（VIO）：
    用摄像头定期修正IMU漂移
    用IMU填补摄像头帧间信息
```

**ML背景学员**：多传感融合在技术上是一个状态估计问题（贝叶斯滤波框架），现在也有端到端学习方法（用神经网络直接融合多传感器数据）。

### 2.5 点云处理：PointNet

**点云（Point Cloud）是什么？**

```
RGB-D摄像头/LiDAR输出 → 三维点的集合
每个点包含：(x, y, z) + 可选 (r, g, b, intensity)
典型数量：1k - 100k 个点

特点：
✓ 无序集合（没有网格结构）
✓ 点的数量可变
✓ 旋转/平移不变性重要
```

**PointNet（斯坦福 2017）的核心思想：**

```
输入：N × 3 点云
    ↓
T-Net（输入变换网络）：学习点云的规范化变换
    ↓
每个点独立处理（Shared MLP）：N × 64
    ↓
T-Net（特征变换网络）：64维特征空间对齐
    ↓
每个点独立处理（Shared MLP）：N × 1024
    ↓
全局最大池化（Max Pooling）：1 × 1024（全局特征）
    ↓
分类/分割头
```

**为什么Max Pooling是关键？**
- 点云是无序集合，需要一个对点排列顺序不变的操作
- 最大池化：f(x₁, x₂, ..., xₙ) = max(x₁, x₂, ..., xₙ)，天然排列不变

**在具身智能中的应用：**
- 物体姿态估计（6D Pose Estimation）
- 场景理解（哪里可以放置物体）
- 接触点预测（在哪抓取）

### 2.6 面试考点

**Q：触觉传感和视觉传感在抓取任务中各自的作用是什么？**

核心区别：
- 视觉：**预测**抓取（哪里抓、怎么抓）
- 触觉：**验证**抓取（是否抓稳、力度是否合适）

一个好的抓取系统需要两者：视觉确定抓取点，触觉提供实时反馈调整。

**Q：PointNet如何处理点云的排列不变性？**

关键技术：
1. 每个点独立处理（无顺序依赖）
2. 全局Max Pooling（天然排列不变）
3. T-Net变换（学习对齐变换）

**Q：什么是手眼标定（Hand-Eye Calibration）？为什么重要？**

手眼标定：确定摄像头坐标系和机器人末端执行器坐标系之间的变换关系。

重要性：机器人看到物体后，必须知道如何把"摄像头坐标中的物体位置"转换为"机器人关节空间的运动指令"，这需要精确的手眼标定。

## Part 3：代码实践

### 目标：PointNet简单实现（分类任务）

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class TNet(nn.Module):
    """T-Net：学习输入变换矩阵"""

    def __init__(self, k=3):
        super(TNet, self).__init__()
        self.k = k
        self.conv1 = nn.Conv1d(k, 64, 1)
        self.conv2 = nn.Conv1d(64, 128, 1)
        self.conv3 = nn.Conv1d(128, 1024, 1)
        self.fc1 = nn.Linear(1024, 512)
        self.fc2 = nn.Linear(512, 256)
        self.fc3 = nn.Linear(256, k * k)

        self.bn1 = nn.BatchNorm1d(64)
        self.bn2 = nn.BatchNorm1d(128)
        self.bn3 = nn.BatchNorm1d(1024)
        self.bn4 = nn.BatchNorm1d(512)
        self.bn5 = nn.BatchNorm1d(256)

    def forward(self, x):
        batchsize = x.size()[0]
        x = F.relu(self.bn1(self.conv1(x)))
        x = F.relu(self.bn2(self.conv2(x)))
        x = F.relu(self.bn3(self.conv3(x)))
        x = torch.max(x, 2, keepdim=True)[0]
        x = x.view(-1, 1024)
        x = F.relu(self.bn4(self.fc1(x)))
        x = F.relu(self.bn5(self.fc2(x)))
        x = self.fc3(x)

        # 初始化为单位矩阵（关键技巧）
        iden = torch.eye(self.k, dtype=torch.float32).view(1, self.k * self.k)
        iden = iden.repeat(batchsize, 1)
        if x.is_cuda:
            iden = iden.cuda()
        x = x + iden
        x = x.view(-1, self.k, self.k)
        return x


class PointNet(nn.Module):
    """PointNet 分类网络"""

    def __init__(self, num_classes=40):
        super(PointNet, self).__init__()

        self.stn = TNet(k=3)          # 输入变换（3D→3D）
        self.fstn = TNet(k=64)        # 特征变换（64D→64D）

        # 点特征提取（Shared MLP）
        self.conv1 = nn.Conv1d(3, 64, 1)
        self.conv2 = nn.Conv1d(64, 64, 1)
        self.conv3 = nn.Conv1d(64, 64, 1)
        self.conv4 = nn.Conv1d(64, 128, 1)
        self.conv5 = nn.Conv1d(128, 1024, 1)

        # 批归一化
        self.bn1 = nn.BatchNorm1d(64)
        self.bn2 = nn.BatchNorm1d(64)
        self.bn3 = nn.BatchNorm1d(64)
        self.bn4 = nn.BatchNorm1d(128)
        self.bn5 = nn.BatchNorm1d(1024)

        # 分类头
        self.fc1 = nn.Linear(1024, 512)
        self.fc2 = nn.Linear(512, 256)
        self.fc3 = nn.Linear(256, num_classes)
        self.bn6 = nn.BatchNorm1d(512)
        self.bn7 = nn.BatchNorm1d(256)
        self.dropout = nn.Dropout(p=0.3)

    def forward(self, x):
        """
        Args:
            x: 点云，形状 (batch_size, 3, num_points)
        Returns:
            output: 分类logits，形状 (batch_size, num_classes)
        """
        n_pts = x.size()[2]

        # 输入变换
        trans = self.stn(x)
        x = x.transpose(2, 1)       # (B, N, 3)
        x = torch.bmm(x, trans)     # 矩阵乘法（批量）
        x = x.transpose(2, 1)       # (B, 3, N)

        # 第一阶段特征提取
        x = F.relu(self.bn1(self.conv1(x)))
        x = F.relu(self.bn2(self.conv2(x)))

        # 特征变换
        trans_feat = self.fstn(x)
        x = x.transpose(2, 1)
        x = torch.bmm(x, trans_feat)
        x = x.transpose(2, 1)

        # 第二阶段特征提取
        x = F.relu(self.bn3(self.conv3(x)))
        x = F.relu(self.bn4(self.conv4(x)))
        x = F.relu(self.bn5(self.conv5(x)))

        # 全局最大池化（核心！排列不变性来自这里）
        x = torch.max(x, 2, keepdim=False)[0]  # (B, 1024)

        # 分类
        x = F.relu(self.bn6(self.fc1(x)))
        x = F.relu(self.bn7(self.dropout(self.fc2(x))))
        x = self.fc3(x)

        return F.log_softmax(x, dim=-1)


# 测试
if __name__ == "__main__":
    # 模拟输入：batch_size=4, 3维坐标, 1024个点
    point_cloud = torch.randn(4, 3, 1024)

    model = PointNet(num_classes=40)
    output = model(point_cloud)

    print(f"输入形状: {point_cloud.shape}")
    print(f"输出形状: {output.shape}")   # 应该是 (4, 40)
    print(f"参数总量: {sum(p.numel() for p in model.parameters()):,}")
```

## Part 4：产品视角

### 传感器能力边界与产品决策

**选择传感器的关键问题：**

| 决策维度 | 考量要点 |
|----------|----------|
| 精度需求 | 任务是否需要亚毫米精度？ |
| 实时性要求 | 控制频率是多少？（传感器延迟必须<控制周期） |
| 成本约束 | LiDAR $2000+ vs RGB摄像头 $50 |
| 环境适应性 | 光照变化/粉尘/高温等 |
| 维护成本 | 触觉传感器耐用性通常较低 |

**典型产品场景的传感器选型：**

| 场景 | 推荐传感器组合 | 原因 |
|------|--------------|------|
| 仓储分拣 | RGB-D + 力传感器 | 物体多样，需要简单力反馈 |
| 精密装配 | RGB-D + 触觉阵列 + 力矩传感器 | 需要精确接触感知 |
| 导览服务 | LiDAR + RGB | 导航主导，不需要操作 |
| 外科手术 | 力传感器 + 视觉（内窥镜） | 安全至关重要，触觉反馈必须 |
